Match Case

Fields may contain a percent character (%), which matches zero or more arbitrary characters. A token and annotation may be entered in the Token_Annotation field separated by an undescore (_), and if an undescore is not present, any annotation will match. A subcorpus name may be entered in the Corpus field after a slash (/). If the Match Case box is checked, strings and tokens are matched with case sensitivity.

The token portion of the Token_Annotation field (only) may contain a full Unix regular expression: ^ = beginning of token, $ = end of token, . = any character, [a-zA-Z0-9] (etc) = character class, [^a-zA-Z0-9] (etc) = negation of character class, * = zero or more repeats, and anything else you may know. To match a special character, you must use 2 backslashes (\\), or put it inside character class brackets. Single quotes appearing in an expression must be doubled. A dash in a character class must appear at the end, a right square bracket must appear at the beginning. Characters and character classes are case sensitive only if the Match Case box is checked. The annotation portion of this field is not case sensitive, and may only contain percent character (%) as a wildcard.

Enter a name in the Save to list field and the resulting list will be saved for navigation.
